# Настройка с Mave.digital

**Название:** Влад Слушает

**Описание:** Запаиси для прогулок

## Как это работает с Mave.digital

### Вариант 1: Использование Mave.digital как хостинга (Рекомендуется)

Mave.digital предоставляет:
- ✅ Облачное хранилище для аудиофайлов
- ✅ Готовый RSS фид
- ✅ Веб-интерфейс для управления
- ✅ Аналитику прослушиваний

**Workflow:**

1. **Скачиваем аудио локально:**
   ```bash
   python yt2pod.py "https://www.youtube.com/watch?v=VIDEO_ID"
   ```

2. **Загружаем на Mave.digital:**
   - Заходим на https://mave.digital/admin
   - Выбираем подкаст "Влад Слушает"
   - Нажимаем "Добавить эпизод"
   - Загружаем аудиофайл из `podcast/media/VIDEO_ID.m4a`
   - Заполняем метаданные (название, описание берутся из лога)
   - Публикуем!

3. **Эпизод автоматически появится в фиде:**
   - RSS обновится сам
   - Подписчики увидят новый эпизод

### Вариант 2: Гибридный подход

**Преимущества:**

**Недостатки:**

### Вариант 3: Полностью локальный RSS (альтернатива)

Если хотите полный контроль, можно разместить свой RSS:

1. **Загружаем аудио на свой хостинг** (GitHub Pages, S3, etc.)
2. **Генерируем RSS нашим скриптом**
3. **Используем свой URL вместо Mave.digital**

## Быстрый старт

### Шаг 1: Скачать видео с YouTube

```bash
# Одно видео
python yt2pod.py "https://www.youtube.com/watch?v=dQw4w9WgXcQ"

# Несколько видео
python yt2pod.py \
  "https://www.youtube.com/watch?v=VIDEO1" \
  "https://www.youtube.com/watch?v=VIDEO2"
```

### Шаг 2: Найти скачанные файлы

```bash
ls -lh podcast/media/
```

Вы увидите файлы типа `VIDEO_ID.m4a`

### Шаг 3: Загрузить на Mave.digital

1. Откройте https://mave.digital/admin
2. Войдите в аккаунт
3. Выберите "Влад Слушает"
4. "Добавить эпизод" → Загрузите M4A файл
5. Заполните:
   - **Название:** скопируйте из логов скрипта (там будет "Title: ...")
   - **Описание:** скопируйте из логов (там будет описание видео)
   - **Длительность:** автоматически определится
6. Опубликуйте!

### Шаг 4: Проверить в подкаст-приложении

Откройте подкаст-приложение (Apple Podcasts, Overcast и т.д.) и обновите подкаст "Влад Слушает" - новый эпизод должен появиться!

## FAQ

**Q: Нужно ли менять что-то в Mave.digital?**  
A: Нет, просто загружайте эпизоды через веб-интерфейс.

**Q: Можно ли автоматизировать загрузку?**  
A: Да, если у Mave.digital есть API. Нужно изучить документацию.

**Q: Куда девать локальный RSS файл?**  
A: При использовании Mave.digital он не нужен - они генерируют свой RSS.

**Q: Можно ли использовать оба RSS?**  
A: Теоретически да, но это создаст дубли для подписчиков. Лучше выбрать один.
